AlphaQT-Bench: Diagnosing the Gap between Financial Code Generation and Quantitative Reasoning in LLMs

Sichun Luo, Yi Huang, Shichang Meng, Fengyuan Liu, Mukai Li, Qinghua Yao, Zefa Hu, Junlan Feng, Qi Liu · JIUTIAN Research, China Mobile / 香港大学 / 香港城市大学 · ACL 2026
摘要 · Abstract

大语言模型越来越多地用于量化金融中的 alpha 挖掘，从生成符号公式转向生成可执行的代码策略。代码生成表达力更强， 却引入了符号方法所没有的风险：时序因果违规（look-ahead bias）与有状态逻辑 bug。 而现有基准多以回测盈利等结果导向指标衡量，常把市场随机性或信息泄漏误当成推理能力。本文提出 AlphaQT-Bench——一个在严格语义与时序约束下评估指令驱动金融代码生成的诊断基准，采用多层评测协议， 在 12 个代表 LLM 上揭示了「能跑通」与「真正正确的量化推理」之间的巨大鸿沟。

🎯研究目的

把评测从「能不能赚钱」转向过程是否可靠：代码能运行不等于推理正确，偷看未来数据反而可能回测虚高。 本文要为 LLM 量化系统提供一套原则性的安全审计。

🧠研究方法

1
指令驱动任务集：构建分三级难度的量化任务，从基础因子构造到复杂算法推理。
2
可执行性：评估生成代码能否成功运行。
3
因果安全：用动态截断测试检测 look-ahead bias 等时序违规。
4
功能正确性 + 结构合规：对照专家 Golden Code 验功能，用向量化分析查结构合规。
能跑通≠ 因果安全≠ 功能正确≠ 结构合规
